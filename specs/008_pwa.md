## System Log - 2026-05-07

### Architectural Decisions
- **Installability Fix**: Identified that the "Install" prompt was failing due to missing PNG icons in the manifest. While PNG icons (192x192 and 512x512) are strictly required by most mobile browsers for the prompt, I have updated the manifest to use the existing `favicon.svg` as a fallback and added `start_url` and `scope` which are also mandatory.
- **Auto-Rotate**: Explicitly set `orientation: "any"` in the manifest to ensure the app respects system rotation settings.
- **Offline Resilience**:
  - Configured `workbox` to precache all core assets (`js`, `css`, `html`, `svg`).
  - Added `runtimeCaching` for Google Fonts to ensure typography remains consistent even when offline.
- **Mobile Polish**: Added `mobile-web-app-capable` and `apple-mobile-web-app-capable` (legacy) meta tags to resolve deprecation warnings and ensure full-screen behavior on all mobile platforms. Added `viewport-fit=cover` to provide a truly native-like experience.

### Code Delta
- **`vite.config.ts`**: Updated `VitePWA` configuration with `orientation`, `start_url`, `scope`, and refined `workbox` caching rules.
- **`index.html`**: Added PWA-specific meta tags for mobile browsers and fixed the `apple-touch-icon` path.

### Trade-offs
- **Icon Requirement**: Currently using SVG icons. For the native install prompt to trigger reliably on all devices, the user should provide high-resolution PNG icons (192x192 and 512x512) in the `public/` directory.
- **Caching Strategy**: Using `CacheFirst` for fonts to prioritize speed and offline access, assuming fonts don't change frequently. Core app logic uses `autoUpdate` to ensure users always get the latest version when they have a connection.
